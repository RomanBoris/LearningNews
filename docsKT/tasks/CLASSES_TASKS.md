## 🟢 УРОВЕНЬ 1: БАЗОВЫЕ КЛАССЫ (5 заданий) — ✅ ЗАВЕРШЁН

### **1.1 Класс Book** ✅
Создай класс для книги. У книги должно быть название и количество страниц. Эти данные передаются при создании книги и не меняются. Добавь метод, который выводит информацию о книге в формате: "Название: [название], Страниц: [количество]".

---

### **1.2 Класс Product** ✅
Создай класс для товара. У товара есть название, цена и признак наличия на складе. Сделай так, чтобы можно было создать товар вообще без параметров, и тогда название будет "Товар", цена 0.0, а на складе его нет. Но при желании можно указать любые значения при создании.

---

### **1.3 Класс Rectangle** ✅
Создай класс для прямоугольника. При создании передаются ширина и высота. Добавь два свойства, которые не хранят значения, а вычисляют их каждый раз при обращении: площадь (ширина умножить на высоту) и периметр (сумма всех сторон). Эти свойства должны быть доступны как обычные поля, но внутри используют getter.

---

### **1.4 Класс Temperature** ✅
Создай класс для работы с температурой. Внутри класса храни температуру в градусах Цельсия, но это свойство должно быть приватным. Добавь публичное свойство для работы с температурой в Фаренгейтах. Когда читаешь это свойство - конвертируй Цельсии в Фаренгейты. Когда записываешь - конвертируй Фаренгейты обратно в Цельсии и сохраняй. Формула перевода: F = C × 1.8 + 32, обратная: C = (F - 32) / 1.8.

---

### **1.5 Класс Person** ✅
Создай класс для человека с именем и возрастом. Сделай так, чтобы при установке имени автоматически обрезались пробелы в начале и в конце. При установке возраста проверяй, что он не отрицательный, и если отрицательный - выбрасывай ошибку с сообщением.

## 🟡 УРОВЕНЬ 2: КОНСТРУКТОРЫ И СВОЙСТВА (5 заданий) — ⏳ В ПРОЦЕССЕ (1/5)

### **2.1 Класс BankAccount с init блоком** ✅
Создай класс банковского счёта. При создании передаются номер счёта и баланс. Эти данные не должны меняться после создания. Добавь блок инициализации, который сразу при создании объекта проверяет: номер счёта не должен быть пустым, а баланс не должен быть отрицательным. Если что-то не так - выбрасывай ошибку с понятным сообщением.

---

### **2.2 Класс User с secondary конструктором** ⏸️ СЛЕДУЮЩЕЕ
Создай класс пользователя. Основной конструктор принимает имя и возраст. У класса также есть свойство email, которое по умолчанию пустое. Добавь второй конструктор, который принимает только имя и автоматически устанавливает возраст в 18. Добавь третий конструктор, который принимает имя, возраст и email, и устанавливает все три значения.

---

### **2.3 Класс Point** ⏸️
Создай класс для точки на плоскости с координатами x и y (целые числа). Добавь метод, который вычисляет расстояние от этой точки до начала координат (0, 0). Используй формулу: корень из (x² + y²). Метод должен возвращать результат как дробное число.

---

### **2.4 Класс Circle** ⏸️
Создай класс для круга. Радиус храни в приватном свойстве. Сделай проверку при установке радиуса: он должен быть больше нуля, иначе ошибка. Добавь два вычисляемых свойства (с getter): площадь круга и длина окружности. Площадь = π × радиус². Длина окружности = 2 × π × радиус. Также нужен публичный метод для установки радиуса, так как само свойство приватное.

---

### **2.5 Класс Password** ⏸️
Создай класс для работы с паролем. Пароль храни не в открытом виде, а в виде хеша (это приватное свойство). Добавь метод для установки пароля - он принимает строку пароля и вычисляет от неё hashCode, преобразует в строку и сохраняет. Добавь метод для проверки пароля - он принимает строку, вычисляет её hashCode и сравнивает с сохранённым хешем, возвращает true или false.

## 🟠 УРОВЕНЬ 3: COMPANION OBJECT И МОДИФИКАТОРЫ (5 заданий) — ⏸️

### **3.1 Класс MathUtils с companion object** ⏸️
Создай класс для математических утилит. Внутри класса создай companion object. В нём определи константу PI со значением 3.14159. Добавь функцию, которая возвращает квадрат числа (число умножить на само себя). Добавь функцию, которая проверяет, является ли число простым (делится только на 1 и на само себя). Все функции должны вызываться через имя класса, без создания объекта.

---

### **3.2 Класс Counter с companion object** ⏸️
Создай класс счётчика. В companion object храни приватную переменную, которая подсчитывает, сколько всего объектов этого класса было создано. При создании каждого нового объекта Counter этот счётчик должен увеличиваться на 1 (используй init блок). Добавь в companion object публичную функцию, которая возвращает текущее значение счётчика.

---

### **3.3 Класс Email с приватным конструктором** ⏸️
Создай класс для email адреса. Конструктор должен быть приватным и принимать строку с адресом. В companion object создай фабричный метод create, который принимает строку, проверяет, что она содержит символ @, и если да - создаёт объект Email, если нет - возвращает null. Таким образом, создать невалидный Email будет невозможно.

---

### **3.4 object Database (singleton)** ⏸️
Создай singleton для работы с базой данных. Это должен быть object (не class!). Внутри храни приватную переменную для соединения с БД (строка, nullable, изначально null). Добавь метод connect, который принимает URL и сохраняет его в эту переменную. Добавь метод isConnected, который проверяет, установлено ли соединение (не null). Добавь метод disconnect, который обнуляет соединение.

---

### **3.5 Класс Logger с уровнями логирования** ⏸️
Создай класс для логирования. В companion object определи три константы для уровней логирования: DEBUG = 0, INFO = 1, ERROR = 2. У класса есть приватное свойство текущего уровня (изначально INFO). Добавь метод для изменения уровня. Добавь метод log, который принимает уровень и сообщение. Вывод сообщения происходит только если переданный уровень больше или равен текущему уровню логгера.

---

## 🔴 УРОВЕНЬ 4: ПРОДВИНУТОЕ (2 обязательных + 3 ознакомительных) — ⏸️

### **4.1 Класс Vector с operator overloading** 📎 ОЗНАКОМИТЕЛЬНОЕ
Создай класс для двумерного вектора с координатами x и y (целые числа). Переопредели оператор plus так, чтобы можно было складывать два вектора (складываются соответствующие координаты). Переопредели оператор times для умножения вектора на целое число (скаляр). Переопредели метод toString, чтобы вектор выводился в формате "Vector(x, y)".

---

### **4.2 Класс Outer с вложенным и внутренним классом** 📎 ОЗНАКОМИТЕЛЬНОЕ
Создай класс Outer с каким-нибудь свойством (например, именем). Внутри создай вложенный класс (nested class) с методом info, который выводит что-то вроде "Static nested class". Этот класс НЕ должен иметь доступа к свойствам внешнего класса. Создай также внутренний класс (inner class) с методом info, который ИМЕЕТ доступ к свойствам внешнего класса и выводит их. Обрати внимание на разницу в создании этих классов.

---

### **4.3 Класс Money с infix функцией** 📎 ОЗНАКОМИТЕЛЬНОЕ
Создай класс для денег с суммой и валютой. Создай infix функцию to, которая принимает название новой валюты и возвращает новый объект Money с той же суммой, но другой валютой. Благодаря infix можно будет писать: dollars to "EUR" вместо dollars.to("EUR"). Переопредели toString для красивого вывода.

---

### **4.4 Класс Stack<T> с обобщениями** ⏸️
Создай класс стека с поддержкой любого типа элементов (используй дженерики). Внутри храни элементы в изменяемом списке (приватное свойство). Реализуй метод push для добавления элемента в конец. Реализуй метод pop для извлечения последнего элемента (удаляет и возвращает, если стек пуст - возвращает null). Реализуй метод peek для просмотра последнего элемента без удаления (если пуст - null). Реализуй метод isEmpty для проверки пустоты.

---

### **4.5 data class vs обычный class** ⏸️
Создай data class User с полями id и name. Создай обычный class Product с теми же полями. В main функции создай по два объекта каждого класса с одинаковыми значениями полей. Сравни их через == и выведи через toString(). Обрати внимание на разницу: data class сравнивает по содержимому и имеет красивый toString, обычный class сравнивает по ссылке и toString показывает адрес в памяти.
